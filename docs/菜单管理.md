# 菜单管理需求文档

## 1. 需求概述

菜单管理用于维护系统菜单结构与菜单基础信息，支持菜单树形展示、菜单新增/编辑/删除、批量删除，以及菜单基本信息维护与子菜单管理。

### 1.1 本期范围

| 维度 | 包含 | 不包含 |
|------|------|--------|
| 菜单数据 | 菜单名称、编码、路径、图标、排序、访问类型、父子层级 | 按钮权限、菜单显隐控制、角色授权绑定 |
| 操作 | 新增、编辑、删除（单/批量）、搜索、树浏览 | 拖拽排序、菜单导入/导出、菜单复制 |
| 使用者 | 系统后台管理员 | 普通用户自助菜单配置 |
| 删除策略 | 物理删除，二次确认，有子菜单禁止删除 | 软删除、级联删除 |

### 1.2 后续扩展（本期不做）

- 角色授权与菜单绑定
- 按钮级别权限控制
- 菜单可见性动态控制
- 拖拽排序

## 2. 数据结构概述

### 2.1 菜单核心字段

| 字段 | 类型 | 必填 | 唯一 | 说明 |
|------|------|------|------|------|
| id | 字符串 | 是 | 是 | 主键，后端自动生成 |
| parentId | 字符串 | 否 | 否 | 父菜单ID，null 表示一级菜单 |
| menuName | 字符串 | 是 | 否 | 菜单名称，最大长度 50 字符 |
| menuCode | 字符串 | 是 | 是（全局） | 菜单编码，前端自动生成，不可手动修改 |
| menuPath | 字符串 | 否 | 否 | 菜单路径，如 `/system/menu`，最大长度 200 字符 |
| accessType | 枚举 | 是 | 否 | 访问类型：`LOCAL`（本地）/ `REMOTE`（远程） |
| sortOrder | 整数 | 是 | 否 | 同级排序值，值越小越靠前，默认 0，步长 10 |
| icon | 字符串 | 否 | 否 | 图标标识，存储图标 class 名 |
| children | 数组 | 否 | 否 | 子菜单集合，用于树形结构返回（前端展示用，非持久化字段） |

### 2.2 菜单编码生成规则

- 由前端在新增时自动生成。
- 生成算法：基于菜单名称的拼音首字母 + 时间戳后 6 位，确保全局唯一。
- 生成后不可编辑修改，在表单中灰显展示。

### 2.3 排序值管理

- 新增菜单默认 `sortOrder = 0`，排在当前父级下的最前面。
- 编辑时支持通过增减按钮调整，每次步长为 ±1。
- 删除菜单后，同级其他菜单的排序值不自动重新整理（允许出现空洞排序值）。
- 本期不做拖拽排序。

## 3. 页面结构

菜单管理页采用左右分栏布局：

```
┌──────────────────┬──────────────────────────────────┐
│                  │                                  │
│                  │  ┌─────────────────────────────┐ │
│  - 菜单树（搜索）  │  │ 基本信息 │ 子菜单列表          │ │
│                  │  │ [编辑表单/列表内容]            │ │
│                  │  │                             │ │
│                  │  └─────────────────────────────┘ │
└──────────────────┴──────────────────────────────────┘
```

## 4. 功能需求

### 4.1 左侧菜单树区

#### 4.1.1 搜索功能

- 支持菜单名称关键字过滤。
- 输入后实时过滤菜单树（前端本地过滤，树数据全量加载到前端后过滤）。
- 搜索结果保持树形层级展开状态。
- 支持清空搜索，恢复完整树结构。

#### 4.1.2 菜单树展示

- 默认展示完整菜单层级结构（全量一次性加载，不做懒加载）。
- 点击节点后加载右侧详情/子菜单列表。
- 选中节点高亮显示。
- 根节点为虚拟节点，用于承载一级菜单：
  - 虚拟根节点 ID 约定为固定字符串 `"virtual_root"`，前端构建，不存储到数据库。
  - 默认激活虚拟根节点，右侧只展示子菜单列表 Tab，基本信息 Tab 隐藏。
  - 虚拟根节点不参与编辑、删除、批量删除操作。
- 只展示节点直接子级菜单列表，不展示孙子节点。

#### 4.1.3 菜单树功能

- 支持手动刷新树数据。
- 支持展开/收起节点。
- 树加载时显示 loading 状态。

### 4.2 右侧编辑区

#### 4.2.1 基本信息 Tab

**表单字段：**

| 字段 | 控件 | 规则 |
|------|------|------|
| 菜单名称 | 文本输入 | 必填，最大 50 字符 |
| 菜单编码 | 文本（灰显只读） | 自动生成，不可编辑 |
| 访问类型 | 单选（本地/远程） | 必选 |
| 菜单路径 | 文本输入 | 选填，最大 200 字符 |
| 排序 | 数字输入 + 增减按钮 | 步长 ±1 |
| 图标 | 图标选择器 | 选填，复用 `MgIconPicker` 组件 |

**表单操作：**

- `确定` 按钮：保存菜单基本信息，成功后刷新树和子菜单列表。
- `取消` 按钮：放弃编辑，恢复表单为上次保存的值。

**表单校验：**

- 菜单名称为必填项，未填写或仅空格时 `确定` 按钮禁用。
- 菜单编码在编辑场景下灰显不可修改。

**编辑规则：**

- 基本信息 Tab 支持直接编辑当前选中菜单，编辑能力与编辑菜单页面保持一致。
- 选中虚拟根节点时，基本信息 Tab 隐藏。
- 访问类型包含本地、远程两种选项，本期不根据访问类型对菜单路径做差异化校验。
- 取消编辑时，已修改数据不保存、不提示。

#### 4.2.2 子菜单列表 Tab

**查询区：**

- 查询条件：`菜单名称`（模糊匹配）、`菜单编码`（精确匹配）。
- 操作：`查询` 按钮、`重置` 按钮（清空查询条件并恢复默认列表）。
- 子菜单列表只展示当前选中菜单的直接子菜单。
- 选中虚拟根节点时，子菜单列表展示一级菜单（parentId 为 null 的菜单）。
- 查询仅在当前选中节点的直接子菜单范围内生效。

**工具区：**

- `新增` 按钮：打开新增菜单页面。
- `批量删除` 按钮：
  - 无勾选时禁用。
  - 已勾选时启用，并显示"已选 N 条"。
  - 二次确认后执行删除。

**列表区：**

| 列字段 | 说明 |
|--------|------|
| 序号 | 根据排序值升序排列的序号 |
| 菜单名称 | 菜单名称 |
| 菜单编码 | 全局唯一编码 |
| 菜单路径 | 菜单路径 |
| 图标 | 图标展示 |
| 操作 | `编辑`、`删除` 按钮 |

- 支持复选框多选（含表头全选）。
- 分页展示，默认每页 10 条。
- 按 `sortOrder` 升序排列。

**列表交互：**

- 编辑：打开编辑菜单页面，修改菜单信息。
- 删除：单行删除，需二次确认弹窗。

### 4.3 新增菜单页面

- 点击新增按钮后，打开新增菜单页面（弹窗或新页面，具体交互由设计阶段确定）。
- 表单字段与基本信息 Tab 一致。
- 菜单编码不显示（前端自动生成）。
- `重置` 按钮：清空表单所有已填内容。
- `确定` 按钮：保存新增子菜单，校验与基本信息 Tab 一致。
- 新增菜单时，传递父菜单 ID：
  - 从虚拟根节点新增时，parentId 为 null → 新增结果为一级菜单。
  - 从真实菜单节点新增时，parentId 为该节点 ID → 新增结果为其子菜单。
- 新增成功后，关闭新增页面，返回主页面并刷新菜单树和子菜单列表，新菜单按 `sortOrder = 0` 展示在父菜单下最前面。

### 4.4 编辑菜单页面

- 点击编辑按钮后，打开编辑菜单页面（弹窗或新页面，具体交互由设计阶段确定）。
- 表单字段与基本信息 Tab 一致，菜单编码灰显不可编辑。
- 编辑菜单时，传递菜单 ID。
- 编辑菜单页面能力与基本信息 Tab 编辑能力完全一致。
- 编辑成功后，关闭编辑页面，返回主页面并刷新菜单树和子菜单列表。

## 5. 业务规则

### 5.1 菜单名称

- 必填，不可为空或纯空格，最大长度 50 字符。
- 同一父级下菜单名称建议不重复（软约束，不强制校验）。

### 5.2 菜单编码

- 由前端在新增时根据菜单名称自动生成。
- 全局唯一，不可手动修改。
- 生成规则：拼音首字母 + 时间戳后 6 位。

### 5.3 菜单路径

- 选填，最大长度 200 字符。
- 本期不根据访问类型做差异化校验。

### 5.4 排序

- 新增菜单默认 `sortOrder = 0`，同级下值越小越靠前。
- 编辑时支持 ±1 步长调整。
- 删除菜单后，同级其他菜单排序值不自动重新整理。

### 5.5 删除规则

- 删除为物理删除（直接从数据库移除）。
- 删除操作（单条和批量）均需二次确认弹窗。
- 删除菜单时，若存在子菜单（`parentId` 等于当前菜单 ID），提示"该菜单下存在子菜单，不允许删除"，操作取消。
- 批量删除时，若任一已勾选菜单存在子菜单，则本次批量删除整体失败，不做部分删除，并提示"所选菜单中存在包含子菜单的菜单，批量删除失败"。
- 批量删除仅对已勾选数据生效；无勾选时按钮禁用。

### 5.6 虚拟根节点

- 虚拟根节点 ID 约定为固定字符串 `"virtual_root"`，仅前端使用，不存储到数据库。
- 虚拟根节点不参与新增、编辑、删除、批量删除。
- 选中虚拟根节点时：
  - 基本信息 Tab 隐藏。
  - 子菜单列表展示一级菜单（parentId 为 null）。
  - 查询一级菜单时，后端接口入参 parentId 不传或传 null。

### 5.7 数据一致性

- 菜单树与子菜单列表数据需保持一致。
- 新增/编辑/删除操作成功后，同步刷新菜单树和当前子菜单列表。

### 5.8 并发编辑

- 本期不做并发编辑冲突处理（如乐观锁）。若多人同时编辑同一菜单，以后提交者覆盖前提交者数据，此风险已知暂不处理。

### 5.9 审计

- 菜单数据需要记录创建人、创建时间、更新人、更新时间（由后端框架审计字段自动维护）。

### 5.10 权限

- 本期菜单管理页面通过菜单入口控制访问权限，不做更细粒度的按钮/操作权限控制。

## 6. 验收标准

### 6.1 菜单树

- [ ] 左侧菜单树可通过菜单名称关键字实时过滤，搜索结果保持层级展开。
- [ ] 默认选中虚拟根节点，并在右侧子菜单列表展示所有一级菜单。
- [ ] 点击真实菜单树节点后，右侧详情能正确加载对应菜单的基本信息和子菜单列表。
- [ ] 支持树节点展开/收起、手动刷新。
- [ ] 树加载时显示 loading 状态。

### 6.2 基本信息编辑

- [ ] 基本信息 Tab 可完成菜单基本信息编辑、保存、表单校验，能力与编辑菜单页面一致。
- [ ] 菜单名称为空时，确定按钮禁用。
- [ ] 菜单编码灰显不可编辑。
- [ ] 选中虚拟根节点时，基本信息 Tab 隐藏。

### 6.3 子菜单列表

- [ ] 子菜单列表 Tab 可通过菜单名称、菜单编码查询当前选中菜单的直接子菜单。
- [ ] 支持分页展示，按排序值升序排列。
- [ ] 支持复选框多选和表头全选。
- [ ] 选中虚拟根节点时展示一级菜单。

### 6.4 新增

- [ ] 可完成子菜单新增闭环：打开新增页面 → 填写表单 → 保存 → 返回主页面并刷新。
- [ ] 新增菜单时能根据菜单名称自动生成全局唯一菜单编码。
- [ ] 从虚拟根节点新增时生成一级菜单。
- [ ] 新增后菜单按 sortOrder = 0 排在父级最前面。

### 6.5 编辑

- [ ] 可完成菜单编辑闭环：打开编辑页面 → 修改表单 → 保存 → 返回主页面并刷新。
- [ ] 编辑页面能力与基本信息 Tab 编辑能力一致。

### 6.6 删除

- [ ] 单条删除有二次确认弹窗。
- [ ] 存在子菜单的菜单删除时提示不允许删除，操作被阻止。
- [ ] 批量删除：无勾选时按钮禁用；有勾选时显示选中条数。
- [ ] 批量删除遇到任一存在子菜单的菜单时整体失败，不做部分删除，并给出明确提示。

### 6.7 数据一致性

- [ ] 菜单树与子菜单列表数据展示一致。
- [ ] 新增/编辑/删除操作成功后，菜单树和子菜单列表同步刷新。

### 6.8 UI/UX

- [ ] 布局与设计规范一致，符合用户习惯。
- [ ] 表单校验提示清晰、友好。
- [ ] 页面交互响应及时，无明显卡顿。

## 7. 非功能要求

- 页面交互响应及时，无明显卡顿。
- 表单校验与删除确认提示清晰、友好。
- 树形数据加载时显示 loading 状态。
- 列表与树形数据展示一致，无明显错位或脏数据。
- 菜单编码生成性能满足正常使用（生成时间 < 10ms）。
- 菜单数据需记录审计字段（创建人、创建时间、更新人、更新时间）。
